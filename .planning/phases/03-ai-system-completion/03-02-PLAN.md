    **index.ts — Rate Limiter in customHonoApps aktivieren:**
    `defineServer()` gibt `{ idleTimeout, port, fetch }` zurueck — KEINE app-Instanz. Daher kann der Rate Limiter NICHT nachtraeglich auf die Top-Level Hono-App angehaengt werden. Stattdessen den Rate Limiter innerhalb der bestehenden `customHonoApps`-Callbacks aktivieren, die Zugriff auf die sub-Hono-App haben.

    Den letzten customHonoApps-Eintrag (AI Routes, ca. Zeile 245-249) aendern:
    ```typescript
    // AI Routes (mit Rate Limiter per D-04)
    {
      baseRoute: "",
      app: (app: any) => {
        const aiRoutes = new Hono();
        // Sensitive Rate Limiter auf alle AI- und Approval-Endpoints
        aiRoutes.use("/*", sensitiveRateLimiter);
        aiRoutes.route("/", aiSystem.chatRoutes);
        const approvalRoutes = new Hono();
        approvalRoutes.use("/*", sensitiveRateLimiter);
        approvalRoutes.route("/", aiSystem.approvalRoutes);
        app.route("/api/v1/ai", aiRoutes);
        app.route("/api/v1/ai/approvals", approvalRoutes);
      },
    },
    ```
    Der `sensitiveRateLimiter` (10 Requests/Minute per D-05) wird so auf alle Chat- und Approval-Requests angewendet. Der Key ist `userId:routePattern`, d.h. verschiedene User haben unabhaengige Limits.
