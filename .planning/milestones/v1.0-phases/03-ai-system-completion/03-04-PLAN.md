  <action>
    **Problem:** `plugin.ts` line 40-43 returns `createMcRoutes` directly as `routes`. But `createMcRoutes` has signature `(deps: McRouteDeps) => Hono` while the framework expects `(app: any) => void`. When the framework calls `routes(honoApp)`, the Hono app is passed as `deps` parameter — `deps.agentService` is undefined, and the returned Hono app with actual routes is discarded.

    **Fix in plugin.ts:**
    1. Change the `routes` getter to return an adapter function with correct signature `(app: Hono) => void`
    2. Inside the adapter, construct stub/minimal McRouteDeps (same pattern as index.ts standalone mode uses)
    3. Call `createMcRoutes(deps)` to get the configured Hono sub-app
    4. Mount the sub-app onto the passed-in `app` using `app.route('/', mcApp)`

    The deps construction should use minimal stubs for now (same as standalone index.ts pattern with `select: async () => []`, etc.). The real deps injection will be solved properly when the module is registered in the main app's index.ts — there, the real services with DB access are available. For the plugin's routes getter, the approach is:

    ```typescript
    get routes() {
      return (app: any) => {
        // Stub-deps fuer den Fall dass das Plugin direkt registriert wird
        // ohne explizite deps-Injection. Im integrierten Modus werden
        // die echten Services ueber index.ts bereitgestellt.
        const agentService = createAgentSessionService({
          insert: async (data) => ({ id: (data.id as string) ?? 'stub' }),
          update: async () => {},
          select: async () => [],
          broadcast: () => {},
        });
        const auditService = createAuditLogService({
          insert: async () => {},
          select: async () => [],
        });
        const mcApp = createMcRoutes({
          agentService,
          auditService,
          queryCosts: async () => ({ costs: [], totalUsd: 0 }),
          checkDatabase: async () => true,
          getActiveAgentCount: async () => 0,
        });
        app.route('/', mcApp);
      };
    }
    ```

    This ensures:
    - The routes getter satisfies `(app: any) => void` contract
    - The framework can call `routes(honoApp)` and routes get mounted onto honoApp
    - The returned value is void (not a Hono app that gets discarded)

    Also add necessary imports at top: `createAgentSessionService` and `createAuditLogService` from services.

    **Important:** Keep the named exports at the bottom unchanged (`export { createMcRoutes as routes }`) — those are for direct import usage, not for plugin registration.
